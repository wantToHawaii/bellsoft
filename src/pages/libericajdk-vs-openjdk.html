Overview

There are multiple OpenJDK options, and it’s often hard to choose one that will suit your organization. To make it easier, this page gives an extensive comparison of Liberica JDK and AdoptOpenJDK (now rebranding as “Eclipse Adoptium”), two of the most popular distributions.

Liberica JDK

Liberica JDK is a free cloud-oriented OpenJDK implementation for modern Java deployments. It aids in development on the broadest number of configurations, good for building microservice architecture and keeping software secure. It includes the HotSpot JVM, which guarantees Oracle certification. Liberica JDK is also known as a human-oriented Java runtime backed by professional engineers.

This distribution is produced and supported by BellSoft, a company that has established itself in the OpenJDK community. As the sole developer and decision-maker, BellSoft is fully responsible for the product and all the updated versions. Its engineers are recognized for having 15+ years of experience with Java plus working previously with Sun Microsystems and Oracle. The key people in the company are members of the OpenJDK Vulnerability Group2 and Java Community Process Executive Committee.3

AdoptOpenJDK

AdoptOpenJDK uses its own build infrastructure and scripts to make prebuilt binaries from the OpenJDK source code. They consist of class libraries and a Java virtual machine, either HotSpot or the Eclipse OpenJ9.

The project is defined as community-led, where “community” means the AdoptOpenJDK community rather than purely OpenJDK. It is run by the Technical Steering Committee (TSC), contributed to by individuals and a handful of sponsors such as IBM, Red Hat, and Microsoft. Before starting to join the Eclipse Foundation in June 2019 under the name of Eclipse Adoptium,4 it was considered the go-to build for migration with more than 20 million downloads. However, as of May 2021, AdoptOpenJDK is still undergoing the transition to Adoptium.

Official Oracle Certification

Liberica JDK is verified for compliance with Java SE specifications by Oracle’s Technology Compatibility Kit (TCK). It ensures that the Java runtime conforms to the standard and that your company can migrate without problems.

Leading OpenJDK Contributor

BellSoft is consistently fixing issues in the OpenJDK mainline branch. Apart from patches, the company contributed one Java Enhancement Proposal (JEP) to the current JDK, adding a whole new feature to the Java™ language.

Protect systems

Rest assured about Java security

Investing in a trustworthy Java runtime is one of the very first things a company should do. A safe JDK lets you run the business without ever worrying about the stability of your software development environment.

AdoptOpenJDK is as secure as open source code can be. Its contributors are working together on hunting down, reviewing, and eliminating security issues. The latest OpenJDK versions are much safer than sticking with outdated releases. That being said, remember that AdoptOpenJDK is an open source project, not a product. It is an environment for software development rather than production. A community of individuals cannot ensure releases concurrent with Oracle Java, security updates or bug fixes. Although IBM support is eligible for OpenJDK binaries from the AdoptOpenJDK community, the company offers its own OpenJDK distribution. Until AdoptOpenJDK finishes rebranding to Adoptium, it’s hard to say how it will be supported. You want a reliable runtime to meet your customers’ standards. Not a project in a transition state.

Liberica JDK, in contrast, is a product for which BellSoft has full responsibility. As a leading OpenJDK contributor, this company cares for security, stability, and conformance. The company’s engineers personally fix bugs for the clients and then ensure the corrections will eventually be available to everyone, as they send patches to the mainline branch of the OpenJDK. As mentioned above, BellSoft is part of the Vulnerability Group alongside other major JDK distributors to ensure your critical applications are safe from common vulnerabilities and exposures.

Free up time

Never miss an update

Getting timely releases means being on top of the latest features that aid in Java development. Security updates are equally, if not more, important, especially for your customers’ safety.

Liberica JDK releases concurrently with Oracle Java and never delays updates. BellSoft provides regular Critical Patch Updates (CPUs) in accordance with the quarterly schedule. Feature releases roll out twice a year. It means entirely new JDK versions every March and September plus fixes for known bugs and common vulnerabilities around the 17th day of January, April, July and October.

We rely on the Liberica team’s experience and expertise to provide timely updates for our customers; together, we keep JetBrains Runtime secure and performant.

Konstantin Bulenkov
Head of JetBrains Runtime

AdoptOpenJDK follows the same cadence. However, due to its community-led nature, the project sometimes cannot keep up with the strict deadlines. Delaying security patches even for a day or two might be harmful, especially for organizations. Until your company gets an update, all the networks and desktops are easy to hit and thus in grave danger.

Benefit from extended support

LTS releases are called “long-term” for a reason. Does your enterprise have tens, hundreds, or even thousands of instances? It’s reasonable to choose the vendor that guarantees its Java™ support to last for extra 4–5 years.

AdoptOpenJDK receives commercial support from a third party. This means no guarantees that your JDK version will get supported after the OpenJDK community stops providing updates (the case of Java 6 and 7, non-LTS releases). AdoptOpenJDK will only continue to build binaries for releases as long as the corresponding upstream repository is actively maintained, unless the situation changes in view of Eclipse Adoptium.

Liberica JDK versions are supported longer than most OpenJDK implementations. BellSoft contributes to the mainline JDK branch. That’s why its engineers have a clear understanding of the roadmap. Liberica JDK will support JDK 8, the most prevalent distribution among Java developers, until 2031. While others, including AdoptOpenJDK, revoke support for versions 6 and 7, BellSoft still ensures their work, provided that your company has commercial support. Finally, the company already knows the support timelines for the next LTS releases, 17 and 21.

Unify runtimes

Forget about migration and compatibility issues

You want to be 100% sure that your Java apps will launch and run properly. Besides, when you switch from Oracle Java, easy migration is something you’d expect from your OpenJDK pick.

AdoptOpenJDK is considered a de facto hub for the OpenJDK community to collaborate on. And yet, these binaries are not TCK-verified. Technology Compatibility Kit is a suite of tests for Java provided by Oracle. Without the verification, there is no guarantee for your application to be compliant with Java SE. The move to Adoptium was meant to change that, but this project has decided to produce separate Eclipse Temurin binaries with certain licensing restrictions (the Java SE TCKs will be used solely for implementations that include only HotSpot based JREs and no other JVM). It should be noted that Martijn Verburg, a Project Management Committee member for Eclipse Adoptium, was elected to the Java Community Process Executive Committee (JCP EC), but as a London Java Community representative. So, TCK verification is still pending for the full scope of AdoptOpenJDK binaries.

For Liberica JDK quality testing is among the highest priorities. Apart from undergoing internal QA/QC procedures before every release, Liberica binaries — from version 11 to the latest — are fully ratified in the Java Community Process with TCK for Java SE specifications. This indicates that this runtime is highly performant and will not damage your critical applications. Moreover, BellSoft CTO Aleksei Voitylov holds a ratified seat at the JCP EC, which proves the company’s commitment to the standard and future of Java.

Stability, speed and accuracy are all very important factors for us. Liberica, our main JDK distribution, was onboarded without any hiccups. We are happy with the level of support provided by the BellSoft team.

Yury Vasyutinskiy
Team Lead at FlowTraders

Develop on all platforms with a single JDK

One runtime for servers on Windows, another for new and shiny Apple silicon devices, and one for your cloud nodes... Irritating? That's why you should opt for the JDK with the highest number of supported configurations.

Liberica JDK has one of the widest support ranges, including all present-day architectures (x86 64/32 bit, ARM, PowerPC, SPARC) and the majority of known operating systems. While both OpenJDK distributions support numerous configurations, some unique options for Liberica JDK are the full-fledged support for lightweight Alpine Linux musl, which was recently merged into OpenJDK, and for Windows AArch64 available in JDK 11 and 16. This makes Liberica JDK the Unified Java Runtime: one runtime for cloud, server and desktop use cases.

Every company’s objective is to maximize gains and minimize costs. In today’s world, where cloud-native solutions are inevitably taking over, this applies to cloud resources as well as any other.

Second, you can easily up your business game with Liberica JDK microcontainers built upon lightweight Alpine Linux, cutting down on traffic, storage, and deployment time.
